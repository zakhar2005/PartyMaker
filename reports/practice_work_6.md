# Отчет по практической работе №6
## Тема: Реализация веб-платформы

## 1. Краткое описание платформы и логики интеграции API
Реализована интеграционная веб-платформа PartyMaker, которая объединяет:
- собственное API генерации QR-кода (`POST /api/qr/generate`);
- API одногруппника (через настраиваемый URL `classmate_api_url`);
- локальный mock API (`POST /api/mock/classmate/recommend`) для тестирования без внешнего сервиса.

Ключевая бизнес-логика реализована в `POST /api/platform/integrate`:
1. Платформа принимает входные данные (`title`, `date`, `movie_link`).
2. Генерирует `qr_url` через собственную логику QR API.
3. Передает объединенный payload во внешний API одногруппника.
4. Возвращает пользователю агрегированный результат и диагностическую информацию об ошибках.

## 2. Пользовательский интерфейс
Добавлена отдельная страница интеграции `platform.html`, которая позволяет:
- отправлять запросы на интеграционный endpoint;
- задавать URL API одногруппника;
- видеть полный JSON-ответ платформы (результаты обоих API и ошибки).
