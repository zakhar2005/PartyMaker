# Отчет по практической работе №6
## Тема: Реализация веб-платформы

## 1. Краткое описание платформы и реализованной логики взаимодействия API
Веб-платформа **PartyMaker** реализует пользовательский сценарий коллективного выбора фильма и объединяет два API:
1. **Собственное API** сервиса — генерация QR-кода с итоговой информацией о фильме (`POST /api/qr/generate`).
2. **Внешнее API (в рамках интеграции)** — **TMDB API** для получения данных о фильмах (топ фильмов, жанры, рекомендации).

Реализованная бизнес-логика взаимодействия:
1. Пользователь через веб-интерфейс запускает процесс подбора фильма.
2. Платформа обращается к TMDB API и получает данные о фильмах/жанрах.
3. После согласования итогового фильма платформа формирует payload (`title`, `date`, `movie_link`) и передает его в собственное QR API.
4. Пользователь получает QR-код, который можно открыть с телефона и перейти к информации о выбранном фильме.

Таким образом, TMDB используется как внешний источник контента, а собственное API — как сервис публикации результата выбора.

## 2. Реализация обмена данными и обработка ошибок
На платформе реализован HTTP-обмен в формате JSON между интерфейсом, серверной частью и API:
- взаимодействие UI ↔ backend API;
- взаимодействие backend ↔ TMDB API;
- взаимодействие backend ↔ endpoint генерации QR.

Проверено:
- корректная передача входных параметров (`title`, `date`, `movie_link`);
- корректный формат JSON-ответов;
- обработка ошибок при недоступности внешнего API (возврат диагностического сообщения).

## 3. Пользовательский интерфейс
Интерфейс платформы позволяет:
- отправлять запросы на API;
- наблюдать ответы API и результат работы бизнес-логики;
- визуально контролировать итог интеграции (включая QR-код и данные о фильме).

Интерфейс связан с серверной частью через API-вызовы и предназначен для демонстрации интеграционного сценария в наглядной форме.

## 4. Развёртывание
Платформа запускается как единое веб-приложение с backend/API и frontend интерфейсом.
Для выполнения интеграции требуется корректная настройка ключей TMDB (`TMDB_API_KEY` или `TMDB_ACCESS_TOKEN`).

## 5. Тестирование
В ходе проверки протестированы:
1. Базовый сценарий запроса данных о фильмах через TMDB.
2. Генерация QR-кода для итогового фильма.
3. Поведение при ошибочных входных данных.
4. Поведение при недоступности внешнего API (ошибки сети/доступа).

Результаты тестирования подтверждают корректность обмена данными между компонентами платформы.
