# Отчет по практической работе №6
## Тема: Реализация веб-платформы

## 1. Краткое описание платформы и логики взаимодействия API
Реализована веб-платформа PartyMaker, в которой интегрированы:
- собственное API генерации QR-кода;
- внешнее API TMDB (получение данных о фильмах).

Ключевая бизнес-логика:
1. Пользователь отправляет `title` или `tmdb_id`.
2. Платформа получает фильм из TMDB API.
3. На основе результата TMDB формируется ссылка на фильм.
4. Собственное QR API создает QR-код с итоговыми данными фильма.
5. Пользователь получает объединенный ответ (данные TMDB + QR URL).

## 2. Взаимодействие и интеграция API
Реализован endpoint: `POST /api/platform/integrate`.
- Входные данные: `title` или `tmdb_id`.
- Внутри endpoint:
  - вызов TMDB API (поиск/детали),
  - формирование payload,
  - генерация QR через собственную логику QR API.
- Выход: JSON с полями `tmdb_movie`, `own_api`, `integration_status`.

## 3. Пользовательский интерфейс
Добавлен интерфейс `platform.html`, который позволяет:
- отправить `title`/`tmdb_id`;
- выполнить интеграционный вызов;
- увидеть полный JSON-ответ платформы (включая QR и данные фильма).

## 4. Развёртывание
1. Настроить TMDB ключ (`TMDB_API_KEY` или `TMDB_ACCESS_TOKEN`).
2. Запустить сервер: `python3 server.py`.
3. Открыть:
   - приложение: `http://localhost:4173`
   - интеграция: `http://localhost:4173/platform.html`

## 5. Тестирование
Проведены проверки:
1. Успешная интеграция (`title` -> TMDB -> QR).
2. Успешная интеграция по `tmdb_id`.
3. Ошибка валидации при пустых входных данных.
4. Ошибка внешнего API при недоступности TMDB/ключа.
