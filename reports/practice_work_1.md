## 1. Определение цели сервиса
**Название сервиса:** PartyMaker  
**Тип:** веб-сервис для группового выбора фильма

### Проблема
У каждого участника компании разные вкусы, из-за чего выбор фильма часто затягивается и приводит к спорам.

### Решение
Создать сервис, в котором каждый пользователь указывает любимые фильмы. Система подбирает схожие варианты (по жанрам, режиссеру и другим признакам), после чего участники оценивают кандидатов до достижения общего решения.

### Как работает сервис
1. **Создание комнаты:** один пользователь создает комнату и получает уникальную ссылку/код.
2. **Подключение друзей:** участники переходят по ссылке и присоединяются к комнате.
3. **Ввод предпочтений:** каждый указывает 1–3 любимых фильма, либо выбирает жанры/режиссеров, либо оценивает предложенные системой фильмы.

## 2. Функционал собственного API
### Функция
**Генерация тематического QR-кода**, который ведет на страницу с выбранным фильмом.

### Задача
После выбора фильма группе нужно быстро поделиться итогом: что смотреть, когда смотреть, где смотреть трейлер/карточку фильма.

### Решение
API принимает данные о фильме и возвращает готовый QR-код, который можно сохранить и отправить друзьям.

### Входные параметры
| Параметр    | Тип    | Обязательность | Описание |
|-------------|--------|----------------|----------|
| `title`     | String | Да             | Название фильма |
| `date`      | String | Нет            | Год/дата выхода |
| `movie_link`| String | Да             | Ссылка на фильм (Кинопоиск/TMDB), трейлер (YouTube) или стриминг |

### Результат
- QR-код (изображение или URL на изображение QR-кода), ведущий на страницу с информацией о выбранном фильме.

## 3. Функционал внешнего API
### Выбранный сторонний сервис
**The Movie Database (TMDB) API**

### Входные данные
1. Название фильма, введенное пользователем.
2. ID фильма (если известен).
3. Предпочитаемый язык ответа (например, `ru-RU`).
4. Флаг необходимости данных о стриминговых платформах.

### Выходные данные
#### При успешном запросе
1. Статус: «успех».
2. Данные о фильме:
   - название;
   - год выпуска;
   - рейтинг;
   - жанры;
   - режиссер;
   - ссылка на постер;
   - список похожих фильмов;
   - список стриминговых платформ.

#### При ошибке
1. Статус ошибки.
2. Описание ошибки.
3. Код ошибки (например, «фильм не найден», «сервис недоступен»).

### Где используется
В подсистеме подбора фильмов.

### Логика интеграции
1. Пользователь вводит название фильма.
2. Сервис вызывает TMDB API.
3. Из ответа извлекаются ID, жанры, режиссер и другие признаки.
4. Данные сохраняются в профиле пользователя.
5. Процесс повторяется для всех участников комнаты.
6. Анализируются пересечения интересов (жанры/режиссеры и т.д.).
7. По ID фильмов запрашиваются похожие фильмы.
8. Формируется список кандидатов с описанием и доступностью на платформах.
9. После выбора фильма отображается полная информация и ссылки на просмотр.

**Кратко:** узнаем предпочтения участников → находим общие интересы → предлагаем релевантные фильмы → помогаем принять общее решение.

## 4. Локальный функционал
1. **Управление пользователями и группами**
   - регистрация и вход (email/пароль);
   - создание и управление комнатами.

2. **Работа с предпочтениями и голосованиями**
   - сбор и подсчет голосов за фильмы-кандидаты;
   - хранение предпочтений (жанры, режиссеры, история просмотров);
   - алгоритм матчинга интересов участников.

3. **Интерфейс и взаимодействие**
   - чат комнаты для обсуждения;
   - административная панель для модерации и управления пользователями.
