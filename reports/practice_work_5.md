# Отчет по практической работе №5
## Тема: Документирование веб-сервиса

## Цель
Подключить Swagger/OpenAPI для автоматической документации веб-API и подготовить текстовое описание веб-сервиса.

## Выполненные шаги
1. Добавлен OpenAPI-документ (`/openapi.json`) с описанием endpoint `POST /api/qr/generate`.
2. Подключен Swagger UI (`/docs`) для визуальной проверки документации.
3. Проведена проверка доступности документации в браузере.
4. Подготовлено текстовое описание сервиса в файле `reports/service_description.md`.

## Описание API генерации QR
- **Маршрут:** `POST /api/qr/generate`
- **Вход:**
  - `title` (string, обязательно)
  - `date` (string, опционально)
  - `movie_link` (string, обязательно)
- **Выход:** JSON с `title`, `date`, `movie_link`, `qr_url`.

## Проверка Swagger UI
Swagger UI доступен по адресу: `http://localhost:4173/docs`.
OpenAPI JSON доступен по адресу: `http://localhost:4173/openapi.json`.

## Ссылка на текстовое описание
Текстовое описание размещено в репозитории: `reports/service_description.md`.

## Вывод
В проект успешно добавлена Swagger-документация и подготовлены текстовые материалы, достаточные для передачи разработчикам и пользователям.
