# PartyMaker

MVP сервиса совместного выбора фильма с реальным TMDB API.

## Что исправлено
- Подключение по коду/QR теперь ведёт в **существующее лобби**, а не создаёт новое.
- Фильмы берутся из **реального TMDB API** (`top_rated` + `discover`), без хардкода.
- Постеры берутся из `poster_path` TMDB (`image.tmdb.org`) и корректно отображаются.
- Финальный QR ведёт на `final.html?room=...`, где открывается страница итогового фильма.

## Требования
- Python 3.10+
- TMDB API key (v3)

## Запуск
```bash
cd PartyMaker
export TMDB_API_KEY=ваш_ключ_tmdb
python3 server.py
```
Открыть: `http://localhost:4173`

## Проверка сценария
1. На ПК: введите имя хоста, нажмите «Создать комнату».
2. На телефоне: сканируйте QR комнаты (или откройте ссылку `?room=CODE`).
3. Введите ник и нажмите «Присоединиться».
4. Хост нажимает «Начать голосование».
5. Участники ставят лайки/дизлайки; хост жмёт «Обработать раунд».
6. При единогласном лайке появляется итог и QR финала.
7. Скан финального QR открывает страницу итогового фильма.

## API (локально)
- `POST /api/rooms` — создать комнату
- `POST /api/rooms/{code}/join` — присоединиться
- `GET /api/rooms/{code}` — состояние комнаты
- `POST /api/rooms/{code}/start` — старт голосования
- `POST /api/rooms/{code}/vote` — голос
- `POST /api/rooms/{code}/process` — обработка раунда
